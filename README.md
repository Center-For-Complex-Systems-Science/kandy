# KANDy — Kolmogorov-Arnold Networks for Dynamics

KANDy is a data-driven method for learning the governing equations of dynamical systems
from trajectory data. It is a reimplementation of SINDy in which the sparse regression
step is replaced by a Kolmogorov-Arnold Network and the inputs are augmented with
Koopman-lifted coordinates.

A preprint is available on arXiv. This repository contains:
- `research_code/` — Experimental notebooks and scripts for each benchmark system
- `src/kandy/` — A crewAI multi-agent system that automates the research workflow

---

## Mathematical Background

### Kolmogorov-Arnold Representation Theorem

The Kolmogorov-Arnold Representation Theorem (KAT) states that any continuous function
f(x_1, ..., x_n) admits a decomposition into sums of univariate continuous functions:

```
f(x_1, ..., x_n) = sum_{q=1}^{2n+1} Phi_q( sum_{i=1}^{n} phi_{q,i}(x_i) ).
```

This motivates Kolmogorov-Arnold Networks (KANs), in which each edge of the network
carries a learnable univariate spline activation, replacing the fixed nonlinearities
of standard MLPs.

### Why Standard KANs Cannot Recover Dynamical Systems

**Bilinear Obstruction (Corollary).** There do not exist continuous functions
h, u, v : [0,1] -> R such that

```
x * y = h(u(x) + v(y))    for all (x, y) in [0,1]^2.
```

*Proof sketch.* Suppose such functions exist. Setting y = 0 gives h(u(x) + v(0)) = 0
for all x, forcing h to vanish on the entire interval [u(x) + v(0) : x in [0,1]].
For any small y* > 0 one can find x* such that u(x*) + v(y*) falls in this zero set,
forcing h(u(x*) + v(y*)) = 0. But h(u(x*) + v(y*)) = x* y* > 0 for x* > 0,
a contradiction. Making y* arbitrarily small forces u(0) to take multiple distinct
values, contradicting well-definedness of u. ∎

**Deep KAN Corollary.** Adding layers is equivalent to composing functions. Applying
the above corollary at each layer shows that arbitrarily deep KANs with no width
expansion also cannot represent bilinear terms x*y from raw scalar inputs.

**Non-injectivity Obstruction (Proposition).** Let the true dynamics be s_dot = f(s)
on R^n, observed through z = h(s) in R^d. If there exist two states s_1 ≠ s_2 with
h(s_1) = h(s_2) but (d/dt)h(s_1) ≠ (d/dt)h(s_2), then no single-valued function
F : R^d -> R^d satisfying z_dot = F(z) for all trajectories exists. Consequently,
no deterministic model class (including KANs) can recover the governing equations
from such incomplete observations.

*Example.* For the harmonic oscillator x_dot = v, v_dot = -x observed only through
y = x, two states (x_0, v_1) and (x_0, v_2) with v_1 ≠ v_2 produce the same
observable y = x_0 but different derivatives y_dot = v_1, v_2. Any KAN
KAN_theta(x_0, x_0) = c produces a single value, so it cannot equal both v_1 and v_2.
The obstruction is structural: the observation map h(x, v) = (x, x) collapses distinct
states, and the regression z_dot = F(z) is ill-posed.

## The KANDy Algorithm

### Formulation

Given a Koopman lift phi : X -> X_tilde defined by

```
phi(x_1, ..., x_n) = (theta_1, theta_2, ..., theta_m),
```

KANDy learns the system

```
x_dot = A * Psi(phi(x)),
```

where:
- `Psi(theta) = (psi_1(theta_1), ..., psi_m(theta_m))` is a separable feature map
  with each `psi_i` a univariate function learned as a spline,
- `A` is a learned linear mixing matrix (n x m, where n = dim(x)).

Equivalently, the model factorizes the vector field as

```
x  -->  phi(x) = theta  -->  Psi(theta)  -->  A * Psi(theta) = x_dot.
```

### Structural Properties

1. The system is **nonlinear in x** through the composition Psi(phi(x)).
2. It is **linear in A** — A can be read off from the KAN output layer weights.
3. It is **nonlinear in the spline parameters** defining psi_i.
4. The model is a **shallow network**: one hidden layer of separable spline activations
   followed by a linear output. The KAN is always `width = [m, n]`.

### Separable Dictionary and Cross-Terms

Each observable depends only on one lifted coordinate: `psi_i = psi_i(theta_i)`.
The feature map Psi is therefore separable:

```
Psi(theta) = (psi_1(theta_1), psi_2(theta_2), ..., psi_m(theta_m)).
```

**Consequence:** Cross-interaction terms such as x*y cannot be generated by the KAN
from raw inputs. They must be encoded explicitly inside the lift phi. For example,
the Lorenz system has equations with x*y and x*z. The correct lift is:

```
phi(x, y, z) = (x, y, z, x^2, x*y, x*z, y^2, y*z, z^2).
```

All feature coupling then occurs through the linear mixing matrix A.

### Relation to SINDy and Koopman Methods

| Method | Dictionary | Regression | Evolution space |
|--------|-----------|------------|-----------------|
| SINDy  | Fixed Theta(x) | LASSO (sparse) | Original state x |
| EDMD   | Fixed Psi(x)   | Least-squares  | Lifted state Psi |
| KANDy  | Learned Psi(phi(x)) | KAN | Original state x |

KANDy differs from classical Koopman/EDMD in that it models `x_dot = A * Psi` rather
than `Psi_dot = K * Psi`. It is a feature-based regression in original state space,
not a linear evolution law in lifted space.

## Benchmark Systems

| File | System | Key lift terms |
|------|--------|---------------|
| `henon.py` | Hénon map | x, y, x² |
| `hopf.ipynb` | Hopf fibration (S³ → S²) | Quaternion components |
| `Lorenz (3).ipynb` | Lorenz ODE | x, y, z, xy, xz, y², yz, z² |
| `Inviscid_Burgers (1).ipynb` | Inviscid Burgers (shocks) | u, u_x, u_xx, u² |
| `Inviscd-Burgers-fourier-mode-ics.ipynb` | Burgers + Fourier ICs | Same + Fourier modes |
| `Kuramoto–Sivashinsky (1).ipynb` | Kuramoto-Sivashinsky PDE | u, u_x, u_xx, u_xxxx |
| `Navier-Stokes.ipynb` | 3D incompressible NS | Velocity, vorticity, pressure |

## Research Automation Crew

`src/kandy/` contains a [crewAI](https://crewai.com) multi-agent system that automates
four research tasks using a hierarchical process (PhD Advisor manages workers):

| Task | Agent | Output |
|------|-------|--------|
| Audit experiments for algorithm correctness and reproducibility | PhD Student | `outputs/audit_report.md` |
| Write a correct KANDy experiment for `{system}` | PhD Student | `outputs/experiment_{system}.py` |
| Design and implement the `kandy` Python package | Professional Coder | `outputs/package_design.md` |
| Write a formal peer review of results | Journal Reviewer | `outputs/review.md` |

The **PhD Advisor** (manager) knows the algorithm precisely and enforces that:
- The KAN is always single-layer (`width=[m, n]`).
- All cross-terms are encoded in the Koopman lift phi, not learned from raw inputs.
- Symbolic extraction (auto_symbolic + SymPy) is always performed.

## Installation

Requires Python ≥ 3.11, < 3.14.

```bash
pip install uv
crewai install      # installs dependencies from uv.lock
```

Set your API key in `.env`:
```
MODEL=anthropic/claude-sonnet-4-6
ANTHROPIC_API_KEY=sk-ant-...
```

---

## Running the Crew

Edit `INPUTS['system']` in `src/kandy/main.py`, then:

```bash
crewai run
```

Supported values for `system`:
`Lorenz`, `Henon`, `Burgers`, `Burgers-Fourier`, `Kuramoto-Sivashinsky`, `Navier-Stokes`, `Hopf`

Outputs are written to `outputs/`.
